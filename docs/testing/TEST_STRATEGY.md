# 测试策略

## 测试分层

| 层级 | 覆盖率目标 | 说明 |
|:----:|:---------:|------|
| 单元测试 | 70%+ | 核心函数/类 |
| 集成测试 | 60%+ | 模块间交互 |
| E2E测试 | 50%+ | 关键流程 |

## 测试原则

1. **独立性** - 每个测试独立运行，不依赖其他测试
2. **可重复** - 每次运行结果一致
3. **可读性** - 测试名称清晰描述测试内容
4. **快速** - 单元测试应该毫秒级完成

## 测试覆盖内容

### 必须覆盖
- 核心功能
- 边界条件
- 异常处理

### 可选覆盖
- 内部实现细节
- 次要路径
