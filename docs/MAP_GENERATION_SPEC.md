# 伊甸世界 - 地图生成规范

## 概述

伊甸世界使用基于噪声的程序化地形生成，通过多层噪声叠加生成多样化的地形特征。

## 地图参数

| 参数 | 值 |
|:-----|:---|
| 地图宽度 | 200 格子 |
| 地图高度 | 100 格子 |
| 格子大小 | 64 像素 |
| 世界种子 | 可配置（默认 12345） |

## 地形类型

| 类型 | 英文 | 出现条件 |
|:-----|:-----|:---------|
| 海洋 | ocean | 仅底部 3 行 |
| 海滩 | beach | elevation < 0.20 |
| 草地 | grass | 默认，elevation > 0.45 |
| 平原 | plains | 默认，elevation <= 0.45 |
| 森林 | forest | moisture > 0.6 && elevation > 0.4 |
| 沙漠 | desert | moisture < 0.3 |
| 沼泽 | swamp | moisture > 0.7 && elevation > 0.35 |
| 山地 | mountain | elevation > 0.80 |
| 山丘 | hill | elevation > 0.70 |
| 河流 | river | 专用算法生成 |
| 湖泊 | lake | 专用算法生成 |

## 地形生成规则

### 1. 海洋 (Ocean)
- **位置**：仅在地图底部 3 行 (y >= 97)
- **目的**：形成统一的海洋边界

### 2. 河流 (River)
- **生成方式**：基于噪声的曲线河流
- **位置**：贯穿地图，从顶部到底部
- **宽度**：约 2% 地图宽度
- **算法**：
  ```
  riverX = 0.5 + noise(y * 0.05, 0) * 0.2 - 0.1
  如果 |x/MAP_WIDTH - riverX| < 0.02，则为河流
  ```
- **限制**：只在 y > 5 && y < MAP_HEIGHT - 5 区域生成

### 3. 湖泊 (Lake)
- **生成方式**：基于噪声的圆形湖泊
- **位置**：地图中部的随机位置
- **半径**：约 6% 地图尺寸
- **算法**：
  ```
  lakeX = 0.3 + noise(0, 0) * 0.4
  lakeY = 0.5 + noise(100, 100) * 0.3
  如果 sqrt((x/MAP_WIDTH - lakeX)^2 + (y/MAP_HEIGHT - lakeY)^2) < 0.06，则为湖泊
  ```
- **限制**：只在 y > 10 && y < MAP_HEIGHT - 10 区域生成

### 4. 基础地形
- 使用双层噪声生成：
  - `elevation` 噪声：控制高度
  - `moisture` 噪声：控制湿度
- 噪声频率：elevation 0.04，moisture 0.06

## 物品生成规则

### 森林 (Forest)
- **覆盖率**：每个森林格子 100% 生成树木
- **树种比例**：
  - 果树 (tree)：约 17%
  - 森林树 (forest_tree)：约 83%
- **生成算法**：
  ```
  hash = (x * 12345 + y * 67890 + seed) % 100
  如果 hash < 17，则为果树，否则为森林树
  ```

### 灌木 (Bush)
- **地形**：草地 (grass)
- **覆盖率**：约 5%
- **条件**：`|noise(x * 0.3, y * 0.3)| < 0.05`

### 石头 (Rock)
- **地形**：山地 (mountain) 或 山丘 (hill)
- **覆盖率**：约 8%
- **条件**：`|noise(x * 0.4, y * 0.4)| < 0.08`

### 贝壳 (Shell)
- **地形**：海滩 (beach)
- **覆盖率**：约 2%
- **条件**：`|noise(x * 0.6, y * 0.6)| < 0.02`

## 出生地规则

### 位置选择
角色出生地需要满足以下条件：
1. **地形**：草地或平原
2. **水源**：5 格范围内有河流或湖泊
3. **食物**：8 格范围内有森林或灌木

### 搜索算法
```typescript
for (y = 10; y < 90; y++) {
  for (x = 10; x < 190; x++) {
    if (tile is grass/plains &&
        exists water within 5 tiles &&
        exists food within 8 tiles) {
      return {x, y};  // 找到合适位置
    }
  }
}
```

## 性能优化

### 当前优化
- 使用简化噪声函数代替完整 Simplex 噪声
- 物品只在服务器端生成，客户端同步
- 地形使用 2D 数组格式传输

### 未来优化建议
- 使用 Web Worker 进行地形预生成
- 实现地形 LOD (Level of Detail)
- 添加地形过渡区域
